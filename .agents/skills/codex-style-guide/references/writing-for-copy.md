# Writing for copy

Source: <https://doc.wikimedia.org/codex/latest/style-guide/writing-for-copy.html>

Copy should help people understand the product and reflect the brand
voice. Run new copy past these questions, grouped by principle.

## Content first

- **Is this needed?** Don't use more text than necessary.
- **Is this relevant?** Provide context, explanations, consequences,
  and solutions. Never give an error without an explanation *and* a fix.

## Design for consistency

- **Is this clear?** Use the same terminology in a title and its CTA;
  group related content; consider how text *looks*, not just what it
  says.
- **Is this concise?** Keep messages short and scannable; focus on the
  user's action; don't over-explain or pile on CTAs.
- **Is this consistent?** Reuse terminology across the UI and the brand;
  keep a list of preferred terms for buttons, tabs, etc.

## This is for everyone

- **Accessible:** avoid stereotype-reinforcing words; avoid directional
  descriptors like "below"/"above" (not everyone perceives layout).
- **Inclusive:** use "a user" or a known username with neutral active
  voice; use "they"/"you"/"we" when gender is unknown — don't assume
  pronouns.
- **Translatable:** plain English, short words, simple phrases; avoid
  idioms, slang, jokes, metaphors, pop-culture references.

## Trustworthy yet joyful

- **Trustworthy:** calm, confident tone; don't cause panic.

## Copywriting tips

- Use a **neutral voice**; focus on functionality.
- Use **bare infinitives** for actions: *Upload*, *Publish*, *Share*.
- Keep things **scannable** (lists, consistent titles/subtitles).
- Make **consequences** clear; give actions in order; list error fixes
  by effectiveness.
- **Verbs for actions, nouns for places.**
- **Speak the user's language** — avoid code terms:
  - "invalid" → "incorrect"
  - "input" → "content" / "text"
  - "value" → name, quantity, etc.
- **Choose words carefully** — avoid *please*, *sorry*, *just*,
  *simply* (grating/condescending) and *try* (implies they might fail).
  - Not "Please try simply refreshing the page" → "Refresh the page to
    load the data".
- **Work with visual design** (colour, grouping, spacing, icons) to
  build a messaging hierarchy.
